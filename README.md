# Entropy-Based Selective Step Verification Using Process Reward Models on Small Language Models

**CS 517: Socially Responsible AI | University of Illinois at Chicago | Spring 2026**

## Overview

Process Reward Models (PRMs) verify each reasoning step generated by language models, but verifying every step is computationally expensive. We propose an entropy-based selective verification framework that measures token-level uncertainty during generation and routes only high-entropy (uncertain) steps to the verifier. Low-entropy (confident) steps are skipped, reducing verification cost while maintaining accuracy.

## Key Finding

Across all three models tested on 200 GSM8K problems, **wrong answers consistently exhibit higher entropy than correct answers**, validating entropy as a reliable signal for selective verification.

| Model | Accuracy | Correct Entropy | Wrong Entropy | Entropy Difference |
|-------|----------|----------------|---------------|-------------------|
| Qwen2.5-Math-1.5B | 84% | 0.187 | 0.252 | +0.065 |
| Llama 3.2 3B | 56% | 0.339 | 0.371 | +0.032 |
| Gemma 3 4B | 57% | 0.124 | 0.151 | +0.027 |

## Method

### Entropy Measurement

During generation, we capture the probability distribution at each token position using `model.generate(output_scores=True)` with KV-cache for efficiency. Token-level entropy is computed as:

```
H(t) = -sum(p(x) * log2(p(x)))
```

Token entropies are aggregated per reasoning step to produce step-level uncertainty scores.

### Verification Strategies

We compare three strategies for deciding which steps to send to the PRM:

1. **Full Verification** (baseline): verify every step
2. **Random Verification** (control): verify a random subset of steps
3. **Entropy-Based Selective Verification** (ours): verify only steps with entropy above a threshold

### Verifiers

- **Math-Shepherd PRM** (Qwen2-0.5B fine-tuned): trained PRM for mathematical reasoning (GSM8K)
- **Gemini LLM-as-Judge** (Gemini 2.5 Flash-Lite via Vertex AI): general-purpose step verifier (both domains)

### SRAI Principles

- **Reliability**: entropy provides a measurable confidence signal per reasoning step
- **Interpretability**: step-level verification scores reveal exactly where a model struggles
- **Robustness**: accuracy is maintained under reduced verification budgets
